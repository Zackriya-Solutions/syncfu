# OBSERVATIONS & DECISIONS
- Using single index.html for both windows — frontend reads window label to decide overlay vs main app
- NotificationManager uses IndexMap (ordered) not HashMap for predictable notification ordering
- Zustand stores use readonly types for immutability enforcement
- Tauri API fully mocked in tests — no Tauri runtime needed for frontend testing
- Coverage thresholds set at 80% in vitest config — currently at 97%+
- tauri-plugin-dialog v2 uses `.buttons(MessageDialogButtons::OkCancelCustom(...))` not `.ok_button_label()`
- Tauri v2 split `Manager` and `Emitter` into separate traits — import `Emitter` for `app.emit()`
